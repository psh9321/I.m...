# EC2 
 - 컴퓨터를 빌려서 원격으로 접속해 사용하는 서비스
 - 명령어 입력시 sudo 를 앞에 입력해 관리자 권한으로 명령어를 실행하도록 한다.

## EC2 인스턴스 생성

1. EC2 > 인스턴스 > 인스턴스 시작  


2. 애플리케이션 및 OS 이미지 (Amazon Machine Image) : Ubuntu 로 선택 


3. 인스턴스 유형 : t3.micro 로 선택
 - AWS EC2 에서 빌리는 컴퓨터 1대를 의미


4. 키페어(로그인) : 새 키페어 생성 클릭 (혹은 기존 생성했던 키페어 선택)
 - EC2 컴퓨터에 접근할 때 사용하는 비밀번호의 역할
 - 유출되는 순간 다른 사람들도 내 EC2 컴퓨터에 접근 가능 (절대 보안)


5. 네트워크 생성

 ### 보안그룹 
  - AWS 클라우드에서의 네트워크 보안
  - 인바운드 트래픽 : 외부에서 EC2 로 보내는 어떤 트래픽만 허용하는지 설정할 수 있다.
  - 아웃바운드 트래픽 : EC2 인스턴스에서 외부로 나가는 트래픽을 설정할 수 있다.
  - IP 범위와 PORT 번호를 설정 할 수 있다.
  - 편집 => 인바운드 보안 그룹 규칙 => 22번 port(원격 접속할 때 사용하는 포트), 80번 port(http 로 접속할 때 사용하는 포트), 443(https 로 접속할 때 사용하는 포트) 보안그룹 규칙 추가


6. 스토리지 구성 : 30GIB, gp3 유형 선택
 - EC2 컴퓨터에서의 저장공간 
 - 프리티어는 30GIB 까지 무료 제공


7. 인스턴스 시작 클릭


8. 인스턴스 페이지에서 생성한 인스턴스 상태가 "실행 중" 일때 까지 대기


9. (선택) 네트워크 보안/탄력적 IP > 탄력적 IP 주소 할당 (*비용이 부과됨)
 - 인스턴스를 중지시켰다가 다시 실행하면 IP가 바뀌는데 이때 바뀌지않는 고정 IP 를 할당 받는다.


10. node 설치 (순서대로 입력)
 - https://www.digitalocean.com/community/tutorials/how-to-install-node-js-on-ubuntu-22-04
 - curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.1/install.sh | bash
 - source ~/.bashrc 
 - nvm i v20

11. (선택) EC2 컴퓨터를 한국시간으로 변경 : sudo dpkg-reconfigure tzdata => asia => seoul 로 변경
 - 기본시간이 UTC 시간 이기때문에 필요시 한국시간으로 변경
