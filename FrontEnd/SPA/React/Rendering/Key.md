# Key
 - React가 이전 렌더링 결과와 새로운 렌더링 결과를 비교(Reconciliation)할 때 각 요소를 식별하기 위한 고유한 값
 - 리스트에서 어떤 항목이 추가, 삭제, 변경되었는지 효율적으로 판단하기 위해 사용된다.
 - Key는 형제(Sibling) 요소 사이에서만 고유하면 된다.
 - Key는 컴포넌트의 props로 전달되지 않는다. 

## 사용예시
```javascript
 Array.from({length : 5}).map((_, i) => {
    return <p key={`고유키-${i}`} >키-{i}</p>
 })
```


## 인덱스 번호만을 key 로 사용하는 경우
 - 배열의 순서가 변경되지 않는 정적인 목록에서는 사용할 수 있다.
 - 하지만 추가, 삭제, 정렬이 발생하는 목록에서는 사용을 지양하는 것이 좋다.
 - 요소의 순서가 변경되면 React가 기존 요소를 잘못 식별하여 불필요한 렌더링이나 state가 꼬일 수 있다.


## 같은 위치의 요소의 key가 변경될 경우
 - React는 기존 컴포넌트를 재사용하지 않고 새로운 컴포넌트로 인식하여 기존 컴포넌트를 Unmount하고 새로 Mount한다. 따라서 내부 state도 초기화된다.
