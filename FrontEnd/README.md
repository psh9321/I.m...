# 웹 표준성
 - W3C 등에서 정의한 웹 기술의 표준 규격이다.
 - 운영체제나 브라우저가 달라도 일관되게 동작하도록 HTML, CSS, JavaScript를 작성하는 것을 의미한다.

## 웹표준 검사 페이지
https://validator.w3.org/#validate_by_input

## 웹 표준을 지켜야 하는 이유

1. 브라우저 호환성 
 - Chrome, Safari, Edge, FireFox 등 에서 동일하게 동작할 가능성이 높아진다.
 - 브라우저 마다 구현 차이를 최소화할 수 있다.


2. SEO(검색 엔진 최적화)
 - 시맨틱 태그를 올바르게 사용하면 검색 엔진이 문서 구조를 더 잘 이해할 수 있다.
 - div 만을 사용하기 보다 header, main, section, article, nav, dl 등을 활용하면 크롤링에 도움된다.


3. 접근성 향상 
 - 스크린 리더가 문서 구조를 정확하게 이해할 수 있다.
 - 키보드 사용자도 이용하기 쉬워진다.


4. 유지보수성 향상
 - 의미 있는 마크업 덕분에 다른 개발자가 코드를 이해하기 쉽다.

5. 예측 가능한 동작
 - 잘못된 구조는 브라우저가 자동으로 DOM 을 수정하면서 예상치 못한 결과를 만들 수 있다.
 - React, Next 에서는 SSR/Hydration 문제의 원인이 되기도 한다.

### 웹표준 기본 구조 
```
    <header>
        <nav></nav>
    </header>

    <h1>홈페이지</h1>
    <main>
        <section>
            <h2>섹션</h2>
            <article>
                <h2>콘텐츠박스</h2>
            </article>
        </section>

        <aside></aside>
    </main>

    <footer></footer>
```

### 웹표준 위반 사례 

```
    <span>
        <div>내용</div>
    </span>
    /** span은 인라인 요소이고 div는 블록 요소이므로 올바른 구조가 아님 */
```

```
    <img src="image.png">
    /** img alt(대체텍스트) 누락 */
```

* 
- h1 태그는 현재 페이지의 제목을 위해 필요하고 h2~6 을 포함한 섹션들 보다 먼저 나와야한다.
- section, article 태그는 반드시 name 태그가 필요하다.
```
    <section>
        <h2>배너</h2>
        <article> 
            <h2>콘텐츠 박스</h2>
        </article>
    </section>
```





# 웹 접근성
 - 장애 유무와 관계없이 누구나 웹서비스를 이용할 수 있도록 만드는 것을 의미

## 웹 접근성 이 중요한 이유 

1. 모든 사용자가 서비스를 이용할수있다.
 - 시각장애인 => 스크린리더 사용
 - 손을 사용하기 어려운 사용자 => 키보드만 사용 
 - 저시력 사용자 => 확대 사용
 - 색양 사용자 => 색상만으로 정보를 전달하지 않음

2. SEO 에도 도움이 된다. 
 - 검색 엔진도 HTML 구조를 분석하기 때문에 시멘틱 태그를 올바르게 작성하면 
   내용을 이해하기 쉬워진다.

## 단순히 태그르 맞게 쓰는 것이 아니라 문서를 순서대로 구성하는 것도 중요하다.
 - 읽는 순서 (DOM 순서), 포커스 이동, 키보드 조작 까지 
   고려하는것이 웹접근성의 중요한 요소이다.

*
- 아이콘 만 있는 버튼의 경우 aria-label 을 사용하거나 
  적절한 태그를 시각적으로 숨겨 스크린리더에 노출되도록 한다.
```
    <button aria-label="로그인">
        <icon/>
    </button>

    <button>
        <span style="position: absolute; top: -999%; left: -999%;   width: 0; height: 0; opacity: 0; z-index: -999;">
            닫기
        </span>
        <icon/>
    </button>
```
