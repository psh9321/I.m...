# 이벤트 캡쳐링 (Event Capturing)
 - 브라우저에서 이벤트 가 발생 할 때 바깥 요소 에서 안쪽 요소 로 이벤트 가 전달 되는 흐름
 - 전역 제어가 필요할때 드물게 사용.
 - 부모 에서 자식으로 내려가는 단계에서 실행되는 이벤트 전파 방식 이며 addEventListener 의 세 번째 인자에 true 주면 캡쳐 단계에서 핸들러 실행


## 이벤트 전파 단계
 1. 캡쳐 단계 (Capturing Phase)
 - window ⇒ document ⇒ html ⇒ body ⇒ …타깃요소 직전 까지
 - 바깥 에서 안쪽 으로 내려 오는 단계

 2. 타깃 단계 (Target Phase)
 - 이벤트 가 실제 로 발생한 요소(타깃)에 도착 하는 단계

 3. Bubbling Phase (버블링 단계)
 - 타깃 요소 ⇒ 부모 ⇒ 상위 로 안쪽 에서 바깥 으로 올라 가는 단계


## 캡쳐링 사용 이유

 1. 버블링 보다 먼저 이벤트 를 가로 채고 싶을때
 - 특정 부모 컴포넌트 가 자식 의 클릭보다 먼저 처리 해야 할 로직 이 있는 경우

 2. 버블링 이 막혀 있어도 동작시키고 싶을때
 - 누군가 event.stopPropagation()을 버블링 단계 에서 막았더라도 캡쳐단계에서는 받을 수 있음

 3. 전역 레벨 에서 모든 클릭을 우선 처리 해야 하는 경우
 - UI 프레임워크, 로깅, 모니터링 등


## 예시 흐름
 - document ⇒ div ⇒ button
